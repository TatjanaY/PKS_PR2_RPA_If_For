# Вариант 11

**Тематика:** Онлайн-галерея для художников — анализ активности пользователей и популярности работ

---

## Контекст задачи

Онлайн-галерея позволяет художникам загружать свои работы, отслеживать просмотры, лайки и комментарии, а пользователям — оценивать и сохранять понравившиеся произведения.

Менеджмент сталкивается с типичными проблемами:

* часть работ получает мало просмотров или низкие оценки;
* отдельные художники редко обновляют свои портфолио;
* необходимо выявлять популярные и проблемные работы, а также активных пользователей;
* прогнозировать, какие работы могут стать популярными в будущем.

**Цель кейса:** реализовать RPA-процесс, который автоматически анализирует активность пользователей и просмотры работ, рассчитывает ключевые показатели популярности, классифицирует работы и пользователей по уровню риска невовлечённости, формирует итоговый отчёт в Excel и выводит уведомление в модальном окне.

---

# Задание 1. Анализ активности за январь (базовый уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Вы — аналитик онлайн-галереи. Необходимо оценить активность художников и пользователей за первый месяц года, выявить работы с низкой популярностью и составить управленческую сводку.

---

## 1. Подготовка данных (Excel)

Создать файл `Gallery_Activity_Jan.xlsx` с листом `Works_Jan`.

**Структура таблицы:**

| Поле       | Тип     | Смысл                                    |
| ---------- | ------- | ---------------------------------------- |
| WorkID     | string  | Уникальный идентификатор работы          |
| ArtistName | string  | Имя художника                            |
| UserName   | string  | ФИО пользователя, оставившего отзыв      |
| Views      | decimal | Количество просмотров                    |
| Likes      | decimal | Количество лайков                        |
| Comments   | int     | Количество комментариев                  |
| Rating     | decimal | Средняя оценка работы пользователями 1–5 |
| UploadDate | date    | Дата загрузки работы                     |

**Методический комментарий:** таблица отражает реальные действия пользователей и показатели популярности работ.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик          | Начальное значение | Смысл                           |
| ---------------- | ------------------ | ------------------------------- |
| total_works      | 0                  | Общее количество работ          |
| low_view_works   | 0                  | Работы с просмотрами < 50       |
| low_rating_works | 0                  | Работы с оценкой < 3            |
| commented_works  | 0                  | Работы с комментариями          |
| total_views      | 0                  | Суммарное количество просмотров |
| total_likes      | 0                  | Суммарное количество лайков     |

---

## 3. Преобразование данных (Puzzle RPA, на каждой строке таблицы)

* `views, likes, rating → decimal`
* `comments → int`
* `uploaddate → date`
* `artistname, username → удалить пробелы`
* `workid → верхний регистр`

**Методический комментарий:** корректное приведение типов обеспечивает точность вычислений и корректное сравнение значений.

---

## 4. Расчёты (Puzzle RPA, на каждой строке таблицы)

```
engagement_score = views + likes * 2 + comments * 3
popularity_percent = (engagement_score / max_engagement) * 100
```

* `engagement_score` — интегральный показатель вовлечённости для работы;
* `popularity_percent` — относительная популярность работы по сравнению с самой популярной работой.

---

## 5. Логика классификации работ (Puzzle RPA, на каждой строке таблицы)

**Простые условия:**

```
if views < 50:
    low_view_works += 1
```

```
if rating < 3:
    low_rating_works += 1
```

```
if comments > 0:
    commented_works += 1
```

**Составные условия:**

```
if popularity_percent < 30 OR rating < 3:
    risk_category = "At Risk"
else:
    risk_category = "Engaged"
```

**Вложенные условия:**

```
if popularity_percent < 10 AND rating < 2:
    risk_category = "Critical"
elif popularity_percent < 30 OR rating < 3:
    risk_category = "High Risk"
else:
    risk_category = "Stable"
```

**Методический комментарий:** простые условия фиксируют просмотры, лайки и комментарии, составные — выявляют риск низкой популярности, вложенные — определяют итоговую категорию работы.

---

## 6. Формирование аналитической таблицы (Excel)

Файл `Gallery_Analysis_Result.xlsx`, лист `Analysis_Jan` с полями:

| WorkID | ArtistName | Views | Likes | Comments | Rating | EngagementScore | PopularityPercent | RiskCategory |

---

## 7. Итоговое уведомление (Puzzle RPA, модальное окно)

```
f"""Анализ работ за январь завершён.

Всего работ: {total_works}
С низкими просмотрами: {low_view_works}
С низкими оценками: {low_rating_works}
С комментариями: {commented_works}
Суммарные просмотры: {total_views}
Суммарные лайки: {total_likes}

Отчёт сохранён в Gallery_Analysis_Result.xlsx (лист Analysis_Jan).
"""
```

# Задание 2. Прогнозная оценка работ за февраль (расширенный уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Во втором месяце необходимо оценить потенциальную популярность и риск низкой вовлечённости работ с учётом текущих просмотров, оценок и прогнозируемого интереса пользователей, используя цикл по строкам таблицы.

---

## 1. Подготовка данных (Excel)

Файл `Gallery_Activity_Feb.xlsx`, лист `Works_Feb` с дополнительными полями:

| Поле             | Тип     | Смысл                                    |
| ---------------- | ------- | ---------------------------------------- |
| WorkID           | string  | Уникальный идентификатор работы          |
| ArtistName       | string  | Имя художника                            |
| Views            | decimal | Количество просмотров                    |
| Likes            | decimal | Количество лайков                        |
| Comments         | int     | Количество комментариев                  |
| Rating           | decimal | Средняя оценка работы пользователями 1–5 |
| ForecastInterest | decimal | Прогнозируемый интерес пользователей 1–5 |
| UploadDate       | date    | Дата загрузки работы                     |

**Методический комментарий:** прогнозируемый интерес позволяет оценить будущую вовлечённость.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик             | Начальное значение | Смысл                             |
| ------------------- | ------------------ | --------------------------------- |
| total_works         | 0                  | Общее количество работ            |
| low_view_works      | 0                  | Работы с просмотрами < 50         |
| low_rating_works    | 0                  | Работы с оценкой < 3              |
| commented_works     | 0                  | Работы с комментариями            |
| total_views         | 0                  | Суммарное количество просмотров   |
| total_likes         | 0                  | Суммарное количество лайков       |
| total_forecast_risk | 0                  | Суммарный прогнозный риск (баллы) |

**Методический комментарий:** переменная `total_forecast_risk` суммирует баллы риска каждой работы:

* `Critical` → +3
* `High Risk` → +2
* `At Risk` → +1
* `Stable` → 0

---

## 3. Цикл по строкам таблицы (Puzzle RPA)

На каждой итерации:

1. Привести типы данных:

* `views, likes, rating, forecastinterest → decimal`
* `comments → int`
* `uploaddate → date`
* `artistname, workid → удалить пробелы / верхний регистр`

2. Вычислить показатели:

```
engagement_score = views + likes * 2 + comments * 3
popularity_percent = (engagement_score / max_engagement) * 100
forecast_diff = popularity_percent - (forecastinterest * 20)
```

* `forecast_diff` отражает разницу между фактической популярностью и прогнозируемым интересом.

---

## 4. Многоуровневая логика прогнозного риска (Puzzle RPA, на каждой строке)

```
if popularity_percent < 10 AND rating < 2:
    risk_category = "Critical"
    total_forecast_risk += 3
elif popularity_percent < 30 OR rating < 3:
    risk_category = "High Risk"
    total_forecast_risk += 2
elif popularity_percent < 50 OR forecastinterest < 3:
    risk_category = "At Risk"
    total_forecast_risk += 1
else:
    risk_category = "Stable"
```

**Методический комментарий:** выполняется для каждой строки. Баллы суммируются в `total_forecast_risk`, чтобы оценить общий прогнозный риск низкой вовлечённости на платформе.

---

## 5. Формирование результирующего Excel (Excel)

Файл `Gallery_Analysis_Result.xlsx`, лист `Analysis_Feb` с полями:

| WorkID | ArtistName | Views | Likes | Comments | Rating | EngagementScore | PopularityPercent | ForecastDiff | RiskCategory |

---

## 6. Итоговое модальное уведомление (Puzzle RPA)

```
f"""Прогнозный анализ работ за февраль завершён.

Всего работ: {total_works}
С низкими просмотрами: {low_view_works}
С низкими оценками: {low_rating_works}
С комментариями: {commented_works}
Суммарные просмотры: {total_views}
Суммарные лайки: {total_likes}
Суммарный прогнозный риск вовлечённости: {total_forecast_risk} баллов

Отчёт сохранён в Gallery_Analysis_Result.xlsx (лист Analysis_Feb).
"""
```
