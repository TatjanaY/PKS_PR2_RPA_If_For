# Вариант 9

**Тематика:** Сервис видеоконференций — анализ активности пользователей и качества соединений

---

## Контекст задачи

Компания предоставляет корпоративный сервис видеоконференций для удалённых команд. В сервисе фиксируются подключения, длительность встреч, качество видео и аудио, а также ошибки соединений.

Менеджмент сталкивается с типичными проблемами:

* часть встреч превышает планируемое время;
* отдельные сессии имели низкое качество связи или были прерваны;
* необходимо выявлять сотрудников с высокой частотой проблем и формировать сводку по качеству сервиса;
* отслеживать динамику активности пользователей за период.

**Цель кейса:** реализовать RPA-процесс, который автоматически анализирует логи видеоконференций, рассчитывает ключевые показатели качества, классифицирует сессии по уровню риска и формирует итоговый отчёт в Excel с уведомлением в модальном окне.

---

# Задание 1. Анализ активности пользователей за март (базовый уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Вы — аналитик службы поддержки. Необходимо оценить активность пользователей и качество их соединений за первый месяц весны, выявить сессии с проблемами и составить управленческую сводку.

---

## 1. Подготовка данных (Excel)

Создать файл `VideoCon_Logs_March.xlsx` с листом `Sessions_March`.

**Структура таблицы:**

| Поле             | Тип     | Смысл                                  |
| ---------------- | ------- | -------------------------------------- |
| SessionID        | string  | Уникальный идентификатор сессии        |
| UserName         | string  | ФИО пользователя                       |
| DurationMin      | decimal | Длительность сессии в минутах          |
| PlannedDuration  | decimal | Плановая длительность сессии в минутах |
| VideoQuality     | decimal | Средняя оценка видео 1–5               |
| AudioQuality     | decimal | Средняя оценка аудио 1–5               |
| ConnectionErrors | int     | Количество ошибок соединения           |
| SessionDate      | date    | Дата проведения сессии                 |

**Методический комментарий:** таблица отражает реальные данные использования сервиса. Значения генерируются через DeepSeek для разнообразия длительности, качества и ошибок.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик                 | Начальное значение | Смысл                                         |
| ----------------------- | ------------------ | --------------------------------------------- |
| total_sessions          | 0                  | Общее количество сессий                       |
| over_duration_sessions  | 0                  | Сессии, длительность которых превышает план   |
| low_quality_sessions    | 0                  | Сессии с видео или аудио < 3                  |
| error_sessions          | 0                  | Сессии с ошибками соединения                  |
| total_extra_minutes     | 0                  | Суммарное превышение длительности всех сессий |
| total_connection_errors | 0                  | Суммарное количество ошибок соединения        |

---

## 3. Преобразование данных (Puzzle RPA, на каждой строке таблицы)

* `durationmin, plannedduration → decimal`
* `videoquality, audioquality → decimal`
* `connectionerrors → int`
* `sessiondate → date`
* `username → удалить пробелы`

**Методический комментарий:** корректное приведение типов обеспечивает точность вычислений и корректное сравнение значений.

---

## 4. Расчёты (Puzzle RPA, на каждой строке таблицы)

* `duration_diff = durationmin - plannedduration`
  *превышение длительности сессии в минутах*

* `average_quality = (videoquality + audioquality) / 2`
  *среднее качество сессии*

**Методический комментарий:** эти показатели помогают оценить фактическое выполнение плановой длительности и качество связи.

---

## 5. Логика классификации сессий (Puzzle RPA, на каждой строке таблицы)

**Простые условия:**

* если `durationmin > plannedduration` → `over_duration_sessions += 1`, `total_extra_minutes += duration_diff`
* если `connectionerrors > 0` → `error_sessions += 1`

**Составные условия:**

* если `average_quality < 3 OR connectionerrors > 2` → `low_quality_sessions += 1`

**Вложенные условия:**

* если `average_quality < 2 AND connectionerrors > 5` → категория = "Critical"
* иначе если `average_quality < 3 OR connectionerrors > 2` → категория = "High Risk"
* иначе → категория = "Stable"

**Методический комментарий:** простые условия фиксируют превышение длительности и ошибки, составные — выявляют низкое качество, вложенные — определяют итоговую категорию риска сессии. Выполняется для каждой строки таблицы.

---

## 6. Формирование аналитической таблицы (Excel)

Файл `VideoCon_Analysis_Result.xlsx`, лист `Analysis_March` с полями:

| SessionID | UserName | DurationDiff | AverageQuality | ConnectionErrors | RiskCategory |

---

## 7. Итоговое уведомление (Puzzle RPA, модальное окно)

```
f"""Анализ сессий за март завершён.

Всего сессий: {total_sessions}
Превышающих длительность: {over_duration_sessions}
С низким качеством: {low_quality_sessions}
С ошибками соединения: {error_sessions}
Суммарное превышение длительности: {total_extra_minutes} мин.
Суммарные ошибки соединений: {total_connection_errors}

Отчёт сохранён в VideoCon_Analysis_Result.xlsx (лист Analysis_March).
"""
```

# Задание 2. Прогнозная оценка сессий за апрель (расширенный уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Во втором месяце количество видеосессий увеличилось. Необходимо оценить потенциальные проблемы по каждой сессии с учётом прошлой динамики и прогнозируемого качества соединения, используя цикл по строкам таблицы.

---

## 1. Подготовка данных (Excel)

Файл `VideoCon_Logs_April.xlsx`, лист `Sessions_April` с дополнительными полями:

| Поле             | Тип     | Смысл                                              |
| ---------------- | ------- | -------------------------------------------------- |
| SessionID        | string  | Уникальный идентификатор сессии                    |
| UserName         | string  | ФИО пользователя                                   |
| DurationMin      | decimal | Длительность сессии                                |
| PlannedDuration  | decimal | Плановая длительность                              |
| VideoQuality     | decimal | Средняя оценка видео 1–5                           |
| AudioQuality     | decimal | Средняя оценка аудио 1–5                           |
| ConnectionErrors | int     | Количество ошибок соединения                       |
| SessionDate      | date    | Дата сессии                                        |
| ForecastQuality  | decimal | Прогнозируемое качество соединения на конец месяца |
| Notes            | string  | Комментарии                                        |

**Методический комментарий:** новые поля позволяют строить прогноз и историю качества соединений.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик                 | Начальное значение | Смысл                                        |
| ----------------------- | ------------------ | -------------------------------------------- |
| total_sessions          | 0                  | Общее количество сессий                      |
| over_duration_sessions  | 0                  | Сессии с превышением длительности            |
| low_quality_sessions    | 0                  | Сессии с низким качеством                    |
| error_sessions          | 0                  | Сессии с ошибками соединения                 |
| total_extra_minutes     | 0                  | Суммарное превышение длительности            |
| total_connection_errors | 0                  | Суммарное количество ошибок соединения       |
| total_forecast_risk     | 0                  | Суммарный прогнозный риск по качеству сессий |

---

## 3. Цикл по строкам таблицы (Puzzle RPA)

На каждой итерации:

1. Привести типы данных:

* `durationmin, plannedduration → decimal`
* `videoquality, audioquality, forecastquality → decimal`
* `connectionerrors → int`
* `sessiondate → date`
* `username → удалить пробелы`

2. Вычислить:

* `duration_diff = durationmin - plannedduration`
* `average_quality = (videoquality + audioquality) / 2`
* `forecast_diff = average_quality - forecastquality`

**Методический комментарий:** прогнозное отклонение отражает возможное снижение качества сессии до конца месяца.

---

## 4. Многоуровневая логика прогнозного риска (Puzzle RPA, на каждой строке)

* если `average_quality < 2 OR forecast_quality < 2` → `risk_category = "Critical"`, `low_quality_sessions += 1`

* иначе если `average_quality < 3 OR forecast_quality < 3` → `risk_category = "High Risk"`, `low_quality_sessions += 1`

* иначе → `risk_category = "Stable"`

* если `duration_diff > 0` → `over_duration_sessions += 1`, `total_extra_minutes += duration_diff`

* если `connectionerrors > 0` → `error_sessions += 1`, `total_connection_errors += connectionerrors`

**Методический комментарий:** выполняется для каждой строки, простые условия фиксируют превышение и ошибки, составные — прогнозируют риск, вложенные — определяют итоговую категорию.

---

## 5. Формирование результирующего Excel (Excel)

Файл `VideoCon_Analysis_Result.xlsx`, лист `Analysis_April` с полями:

| SessionID | UserName | DurationDiff | AverageQuality | ForecastDiff | ConnectionErrors | RiskCategory |
