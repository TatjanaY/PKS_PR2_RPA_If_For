# Вариант 10

**Тематика:** Платформа электронных книг — анализ активности пользователей и качества чтения

---

## Контекст задачи

Платформа электронных книг предоставляет пользователям возможность читать книги онлайн и оффлайн, отслеживать прогресс, оставлять заметки и оценки.

Менеджмент сталкивается с типичными проблемами:

* часть пользователей не завершает книги вовремя;
* отдельные книги имеют низкую оценку качества или интереса;
* необходимо выявлять наиболее активных и проблемных пользователей;
* отслеживать динамику чтения и прогнозировать риск снижения вовлечённости.

**Цель кейса:** реализовать RPA-процесс, который автоматически анализирует логи чтения пользователей, рассчитывает ключевые показатели прогресса и интереса, классифицирует книги и пользователей по уровню риска, формирует итоговый отчёт в Excel и выводит уведомление в модальном окне.

---

# Задание 1. Анализ активности пользователей за январь (базовый уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Вы — аналитик продукта платформы. Необходимо оценить активность пользователей и прогресс чтения за первый месяц года, выявить книги с низким вовлечением и составить управленческую сводку.

---

## 1. Подготовка данных (Excel)

Создать файл `Ebook_Reading_Logs_Jan.xlsx` с листом `Reading_Jan`.

**Структура таблицы:**

| Поле        | Тип     | Смысл                                  |
| ----------- | ------- | -------------------------------------- |
| ReadingID   | string  | Уникальный идентификатор сессии чтения |
| UserName    | string  | ФИО пользователя                       |
| BookID      | string  | Идентификатор книги                    |
| PagesRead   | decimal | Количество прочитанных страниц         |
| TotalPages  | decimal | Общее количество страниц книги         |
| Rating      | decimal | Оценка книги пользователем 1–5         |
| Notes       | string  | Заметки пользователя                   |
| ReadingDate | date    | Дата сессии чтения                     |

**Методический комментарий:** таблица отражает реальные данные пользователей. Значения генерируются через DeepSeek для разнообразия количества прочитанных страниц и оценок книг.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик              | Начальное значение | Смысл                                    |
| -------------------- | ------------------ | ---------------------------------------- |
| total_sessions       | 0                  | Общее количество сессий чтения           |
| incomplete_books     | 0                  | Книги, которые не прочитаны полностью    |
| low_rating_books     | 0                  | Книги с оценкой < 3                      |
| user_notes_present   | 0                  | Сессии с заметками                       |
| total_pages_overread | 0                  | Суммарное количество прочитанных страниц |
| total_low_rating     | 0                  | Суммарное количество низкооцененных книг |

---

## 3. Преобразование данных (Puzzle RPA, на каждой строке таблицы)

* `pagesread, totalpages, rating → decimal`
* `readingdate → date`
* `username → удалить пробелы`
* `bookid → верхний регистр`

**Методический комментарий:** корректное приведение типов позволяет точно рассчитывать процент прочтения и корректно сравнивать оценки.

---

## 4. Расчёты (Puzzle RPA, на каждой строке таблицы)

```
pages_remaining = totalpages - pagesread
read_percent = (pagesread / totalpages) * 100
```

* `pages_remaining` — количество страниц, которое осталось прочитать;
* `read_percent` — процент прочтения книги пользователем.

**Методический комментарий:** эти показатели позволяют оценить вовлечённость и прогресс пользователя по каждой книге.

---

## 5. Логика классификации сессий (Puzzle RPA, на каждой строке таблицы)

**Простые условия:**

```
if pagesread < totalpages:
    incomplete_books += 1
```

```
if rating < 3:
    low_rating_books += 1
    total_low_rating += (3 - rating)
```

```
if notes != "":
    user_notes_present += 1
```

**Составные условия:**

```
if read_percent < 50 OR rating < 3:
    risk_category = "At Risk"
else:
    risk_category = "Engaged"
```

**Вложенные условия:**

```
if read_percent < 25 AND rating < 2:
    risk_category = "Critical"
elif read_percent < 50 OR rating < 3:
    risk_category = "High Risk"
else:
    risk_category = "Stable"
```

**Методический комментарий:** простые условия фиксируют факты (неполное чтение, низкие оценки, наличие заметок), составные — выявляют рисковые книги, вложенные — определяют итоговую категорию вовлечённости.

---

## 6. Формирование аналитической таблицы (Excel)

Файл `Ebook_Analysis_Result.xlsx`, лист `Analysis_Jan` с полями:

| ReadingID | UserName | BookID | PagesRemaining | ReadPercent | Rating | RiskCategory | Notes |

---

## 7. Итоговое уведомление (Puzzle RPA, модальное окно)

```
f"""Анализ чтения за январь завершён.

Всего сессий: {total_sessions}

С низкой оценкой: {low_rating_books}
С заметками: {user_notes_present}
Суммарные страницы прочтены: {total_pages_overread}
Суммарные низкие оценки: {total_low_rating}

Отчёт сохранён в Ebook_Analysis_Result.xlsx (лист Analysis_Jan).
"""
```

# Задание 2. Прогнозная оценка чтения за февраль (расширенный уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Во втором месяце необходимо оценить потенциальный риск снижения вовлечённости пользователей с учётом текущего прогресса по книгам и прогнозируемого интереса, используя цикл по строкам таблицы.

---

## 1. Подготовка данных (Excel)

Файл `Ebook_Reading_Logs_Feb.xlsx`, лист `Reading_Feb` с дополнительными полями:

| Поле             | Тип     | Смысл                                           |
| ---------------- | ------- | ----------------------------------------------- |
| ReadingID        | string  | Уникальный идентификатор сессии чтения          |
| UserName         | string  | ФИО пользователя                                |
| BookID           | string  | Идентификатор книги                             |
| PagesRead        | decimal | Количество прочитанных страниц                  |
| TotalPages       | decimal | Общее количество страниц книги                  |
| Rating           | decimal | Оценка книги пользователем 1–5                  |
| ForecastInterest | decimal | Прогнозируемый интерес пользователя к книге 1–5 |
| Notes            | string  | Заметки пользователя                            |
| ReadingDate      | date    | Дата сессии                                     |

**Методический комментарий:** новые поля позволяют оценить прогнозируемый интерес и вовлечённость пользователей.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик              | Начальное значение | Смысл                                    |
| -------------------- | ------------------ | ---------------------------------------- |
| total_sessions       | 0                  | Общее количество сессий                  |
| incomplete_books     | 0                  | Книги, которые не прочитаны полностью    |
| low_rating_books     | 0                  | Книги с оценкой < 3                      |
| user_notes_present   | 0                  | Сессии с заметками                       |
| total_pages_overread | 0                  | Суммарное количество прочитанных страниц |
| total_low_rating     | 0                  | Суммарное количество низкооцененных книг |
| total_forecast_risk  | 0                  | Суммарный прогнозный риск (баллы)        |

**Методический комментарий:** переменная `total_forecast_risk` суммирует баллы риска для каждой сессии:

* `Critical` → +3
* `High Risk` → +2
* `At Risk` → +1
* `Stable` → 0

---

## 3. Цикл по строкам таблицы (Puzzle RPA)

На каждой итерации:

1. Привести типы данных:

* `pagesread, totalpages, rating, forecastinterest → decimal`
* `readingdate → date`
* `username → удалить пробелы`
* `bookid → верхний регистр`

2. Вычислить показатели:

```
pages_remaining = totalpages - pagesread
read_percent = (pagesread / totalpages) * 100
forecast_diff = read_percent - forecastinterest * 20
```

* `forecast_diff` показывает разницу между фактическим прогрессом и прогнозируемым интересом.

---

## 4. Многоуровневая логика прогнозного риска (Puzzle RPA, на каждой строке)

```
if read_percent < 25 AND rating < 2:
    risk_category = "Critical"
    total_forecast_risk += 3
elif read_percent < 50 OR rating < 3:
    risk_category = "High Risk"
    total_forecast_risk += 2
elif read_percent < 75 OR forecastinterest < 3:
    risk_category = "At Risk"
    total_forecast_risk += 1
else:
    risk_category = "Stable"
```

**Методический комментарий:** выполняется для каждой строки. Баллы суммируются в `total_forecast_risk`, чтобы получить общую оценку риска снижения вовлечённости на платформе.

---

## 5. Формирование результирующего Excel (Excel)

Файл `Ebook_Analysis_Result.xlsx`, лист `Analysis_Feb` с полями:

| ReadingID | UserName | BookID | PagesRemaining | ReadPercent | Rating | ForecastDiff | RiskCategory | Notes |

---

## 6. Итоговое модальное уведомление (Puzzle RPA)

```
f"""Прогнозный анализ чтения за февраль завершён.

Всего сессий: {total_sessions}

Суммарные страницы прочтены: {total_pages_overread}
Суммарные низкие оценки: {total_low_rating}
Суммарный прогнозный риск вовлечённости: {total_forecast_risk} баллов

Отчёт сохранён в Ebook_Analysis_Result.xlsx (лист Analysis_Feb).
"""
```
