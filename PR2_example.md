# Вариант 1

---

## Контекст задачи

IT-компания ведёт одновременно 30–50 проектов: веб-разработка, мобильные приложения, бэкенд-сервисы и комплексные цифровые решения. Проекты находятся на разных стадиях жизненного цикла — от инициации до завершения.

Руководство сталкивается с типичными проблемами:

* часть проектов активна, но демонстрирует перерасход бюджета;
* некоторые проекты приближаются к дедлайну, и фактическая динамика расходов указывает на возможный срыв сроков;
* отдельные проекты уже просрочены;
* отсутствует единая классификация уровня риска по портфелю.

**Цель кейса:** реализовать RPA-процесс, который автоматически анализирует портфель проектов, рассчитывает ключевые показатели, классифицирует проекты по уровню риска, формирует итоговый отчёт в Excel и выводит управленческое уведомление в модальном окне.

---

# Задание 1. Анализ портфеля Q1 (базовый уровень)

**Выполняется в Puzzle RPA и Excel.**

**История:** Вы — аналитик PMO компании. Необходимо оценить финансовое и временное состояние проектов за первый квартал, выявить перерасходы и просрочки, а затем сформировать управленческую сводку.

---

## 1. Подготовка данных (Excel)

Создать файл `IT_Project_Lifecycle_Q1.xlsx` с листом `Projects_Q1`.

**Структура таблицы:**

| Поле        | Тип     | Смысл                                         |
| ----------- | ------- | --------------------------------------------- |
| ProjectID   | string  | Уникальный идентификатор проекта              |
| Status      | string  | Статус проекта: New, Active, Completed        |
| Budget      | decimal | Бюджет проекта в тыс. $                       |
| Spent       | decimal | Фактические затраты в тыс. $                  |
| StartDate   | date    | Дата начала проекта                           |
| EndDate     | date    | Дата завершения проекта                       |
| ClientName  | string  | Наименование клиента                          |
| ProjectType | string  | Тип проекта (Web, Mobile, Backend, Fullstack) |

**Методический комментарий:** таблица отражает исходные данные для анализа. Значения генерируются через DeepSeek для разнообразия бюджета, дат и типов проектов.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик             | Начальное значение | Смысл                                                                |
| ------------------- | ------------------ | -------------------------------------------------------------------- |
| total_projects      | 0                  | Общее количество проектов в портфеле                                 |
| new_projects        | 0                  | Количество проектов на стадии инициации                              |
| active_projects     | 0                  | Количество активных проектов                                         |
| completed_projects  | 0                  | Количество завершённых проектов                                      |
| overbudget_projects | 0                  | Количество проектов с перерасходом бюджета                           |
| expired_projects    | 0                  | Количество проектов с просроченной датой завершения                  |
| risk_projects       | 0                  | Количество проектов, попавших в зону риска                           |
| critical_projects   | 0                  | Количество проектов с критическим финансовым и временным отклонением |
| totalFinancialLoad  | 0                  | Суммарный перерасход активных проектов, тыс. $                       |

---

## 3. Преобразование данных (Puzzle RPA, на каждой строке таблицы)

* `budget, spent → decimal`
* `startdate, enddate → date`
* `clientname → удалить пробелы`
* `projecttype → верхний регистр`

**Методический комментарий:** корректное преобразование типов обеспечивает точность вычислений и корректное сравнение дат и чисел в последующих условиях.

---

## 4. Расчёты (Puzzle RPA, на каждой строке таблицы)

* `deviation = spent - budget`
  *абсолютное отклонение бюджета проекта в тыс. $*

* `deviation_percent = (spent - budget) / budget * 100`
  *процентное отклонение бюджета*

* `remaining_days = enddate - currentdate`
  *количество дней до завершения проекта; отрицательное значение → просрочка*

**Методический комментарий:** эти расчёты позволяют оценить финансовую нагрузку и временной риск проекта.

---

## 5. Логика классификации проектов (Puzzle RPA, на каждой строке таблицы)

**Простые условия:**

* если `spent > budget` → `overbudget_projects += 1`
* если `enddate < currentdate` → `expired_projects += 1`

**Составные условия:**

* если `spent > budget AND status = "Active"` → `totalFinancialLoad += deviation`
* если `deviation_percent > 10 OR enddate < currentdate` → `risk_projects += 1`

**Вложенные условия:**

* если `status = "Active"`:

  * если `deviation_percent > 20` → `critical_projects += 1`
  * иначе → проект считается управляемым
* если `status = "Completed"` → `completed_projects += 1`
* если `status = "New"` → `new_projects += 1`

**Методический комментарий:** простые условия фиксируют факты (перерасход, просрочку), составные — финансовую нагрузку и зону риска, вложенные — итоговую категорию проекта. Выполняется для каждой строки таблицы.

---

## 6. Формирование аналитической таблицы (Excel)

Файл `IT_Project_Lifecycle_Result.xlsx`, лист `Analysis_Q1` с полями:

| ProjectID | Deviation | DeviationPercent | RemainingDays | RiskCategory | IsExpired |

---

## 7. Итоговое уведомление (Puzzle RPA, модальное окно)

```
f"""Анализ портфеля Q1 завершён.

Всего проектов: {total_projects}
Активных: {active_projects}
Новых: {new_projects}
Завершённых: {completed_projects}

С перерасходом: {overbudget_projects}
Просроченных: {expired_projects}
Рисковых: {risk_projects}
Критических: {critical_projects}
Суммарная финансовая нагрузка активных проектов: {totalFinancialLoad} тыс. $

Отчёт сохранён в IT_Project_Lifecycle_Result.xlsx (лист Analysis_Q1).
"""
```

# Задание 2. Расширенный анализ Q2 с прогнозной оценкой риска

**Выполняется в Puzzle RPA и Excel.**

**История:** Во втором квартале количество проектов увеличилось. Необходимо оценить уровень риска по каждому проекту с учётом текущей динамики расходов и оставшихся дней, используя цикл по строкам таблицы.

---

## 1. Подготовка данных (Excel)

Файл `IT_Project_Lifecycle_Q2.xlsx`, лист `Projects_Q2` с той же структурой, что и Q1. Данные генерируются через DeepSeek.

---

## 2. Инициализация переменных и счётчиков (Puzzle RPA)

| Счётчик             | Начальное значение | Смысл                                                                |
| ------------------- | ------------------ | -------------------------------------------------------------------- |
| total_projects      | 0                  | Общее количество проектов в портфеле                                 |
| new_projects        | 0                  | Количество проектов на стадии инициации                              |
| active_projects     | 0                  | Количество активных проектов                                         |
| completed_projects  | 0                  | Количество завершённых проектов                                      |
| overbudget_projects | 0                  | Количество проектов с перерасходом бюджета                           |
| expired_projects    | 0                  | Количество проектов с просроченной датой завершения                  |
| risk_projects       | 0                  | Количество проектов, попавших в зону риска                           |
| critical_projects   | 0                  | Количество проектов с критическим финансовым и временным отклонением |
| totalFinancialLoad  | 0                  | Суммарный перерасход активных проектов, тыс. $                       |
| totalForecastRisk   | 0                  | Суммарный прогнозный риск по портфелю                                |

---

## 3. Цикл по строкам таблицы (Puzzle RPA)

На каждой итерации:

1. Привести типы и формат данных:

* `budget, spent → decimal`
* `startdate, enddate → date`
* `clientname → удалить пробелы`
* `projecttype → верхний регистр`

2. Вычислить:

* `deviation = spent - budget`
* `deviation_percent = (spent - budget) / budget * 100`
* `remaining_days = enddate - currentdate`
* `total_duration_days = enddate - startdate`
* `forecast_deviation = deviation_percent * (remaining_days / total_duration_days)`

**Методический комментарий:** прогнозное отклонение отражает потенциальный риск перерасхода до конца проекта.

---

## 4. Многоуровневая логика прогноза риска (Puzzle RPA, на каждой строке)

* если `status = "Active"`:

  * если `enddate < currentdate` (проект просрочен):

    * если `deviation_percent > 20` → `risk_category = "Crisis"`, `critical_projects += 1`
    * иначе → `risk_category = "Delayed"`
  * иначе:

    * если `forecast_deviation > 30` → `risk_category = "Critical"`, `critical_projects += 1`
    * если `forecast_deviation > 20` → `risk_category = "High Risk"`, `risk_projects += 1`
    * если `forecast_deviation > 10` → `risk_category = "Moderate Risk"`, `risk_projects += 1`
    * иначе → `risk_category = "Stable"`
* если `status = "Completed"` → `risk_category = "Closed"`
* если `status = "New"` → `risk_category = "Planning"`

**Методический комментарий:** выполняется для каждой строки, чтобы классифицировать проекты по совокупности финансового и временного показателей. Простые условия фиксируют факты, составные — финансовую нагрузку и зону риска, вложенные — итоговую категорию проекта.

---

## 5. Формирование результирующего Excel (Excel)

Файл `IT_Project_Lifecycle_Result.xlsx`, лист `Analysis_Q2` с полями:

| ProjectID | DeviationPercent | RemainingDays | ForecastDeviation | FinalStatus |

---

## 6. Итоговое модальное уведомление (Puzzle RPA)

```
f"""Расширенный анализ Q2 завершён.

Обработано проектов: {total_projects}
Критических: {critical_projects}
Рисковых: {risk_projects}
Просроченных: {expired_projects}
Суммарная финансовая нагрузка активных проектов: {totalFinancialLoad} тыс. $
Суммарный прогнозный риск портфеля: {totalForecastRisk}%

Отчёт сформирован в IT_Project_Lifecycle_Result.xlsx (лист Analysis_Q2).
"""
```
